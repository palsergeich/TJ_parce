# TJ_parce — рабочее место по производительности 1С на основе техжурнала

Платформа анализа технологического журнала (ТЖ) 1С:Предприятие: сверхбыстрый нормализатор на C++, загрузка в ClickHouse, готовые дашборды Grafana для инженера по производительности.

**Цифры на реальном корпусе** (175 ГБ ТЖ, 7100 файлов, боевой кластер):

| Метрика | Значение |
|---|---|
| Скорость нормализации | до **3.7 ГБ/с** (AVX2 + memory-mapped I/O) |
| Загрузка полного корпуса в ClickHouse | **121 485 342 события за 33 минуты** |
| Место в БД | **5.59 ГиБ** — сжатие **27×** (несжатых данных 152 ГиБ) |
| Валидность выхода | 100% строк — валидный JSON (проверено на 21+ млн строк) |

## Зачем

Штатные средства анализа ТЖ — это grep по гигабайтам текста. Здесь вместо этого:

1. **Нормализатор** превращает сырой ТЖ в NDJSON (одно событие = один JSON-объект со всеми свойствами) на скорости чтения диска.
2. **ClickHouse** хранит события в колоночном виде с типизированными «горячими» полями (пользователь, сеанс, контекст, текст запроса, память, CPU...) и агрегатами (материализованные представления по минутам/контекстам/запросам/блокировкам).
3. **Grafana** отвечает на вопросы инженера: *что тормозит у пользователей, какие запросы жрут СУБД и из какой строки кода 1С они прилетели, кто кого блокирует, куда уходит память rphost*.

## Архитектура

```
[Сервер 1С]                        [Сервер анализа]                [Инженер]
агент-сборщик                   →  ClickHouse (первый бэкенд)   →  Grafana (этап 1)
 ├─ ядро-нормализатор (C++)        PostgreSQL / DuckDB / MSSQL     свой UI (этап 2)
 ├─ онлайн-хвост каталогов ТЖ      (за слоем абстракции Store)
 └─ разовый импорт архива          Prometheus ← /metrics агентов
```

Язык агента-сборщика выбирается **соревнованием Go vs Rust vs C++** на реальном корпусе — протокол в [docs/bakeoff-protocol.md](docs/bakeoff-protocol.md). Первый участник ([agents/go](agents/go)) уже проходит golden-гейт байт-в-байт с эталоном.

## Быстрый старт

Требования: Windows 10/11, Docker Desktop, PowerShell. Для пересборки ядра — VS Build Tools 2022 (C++), для Go-агента — Go 1.22+.

```powershell
# 1. Поднять хранилище и визуализацию (схема БД применится автоматически)
docker compose -f deploy\docker-compose.yml up -d

# 2. Нормализовать каталог с ТЖ в NDJSON
cpp_parse\build\count_contexts.exe D:\Логи\ТЖ 16 events.jsonl

# 3. Загрузить в ClickHouse
powershell -File deploy\importer\import-jsonl.ps1 -Path events.jsonl

# 4. Смотреть
# Grafana:    http://localhost:3000  (admin/admin) → папка «ТехЖурнал»
# ClickHouse: http://localhost:8123 (HTTP), localhost:9001 (native TCP)
```

Подробности, обслуживание и полезные SQL-запросы — в **[руководстве пользователя](docs/user-guide.md)**.

## Дашборды

| Дашборд | Вопрос инженера |
|---|---|
| Производительность: обзор | Насколько занят кластер (среднее число занятых потоков), какая доля времени — СУБД, p50/p95/p99 отклика, ожидания на блокировках |
| Серверные вызовы (CALL) | Что тормозит у пользователей: топ методов/пользователей по времени, распределение длительностей, память и CPU rphost |
| СУБД (DBMSSQL) | Топ запросов по времени, **из какой строки кода 1С рождается время СУБД**, долгие запросы с текстом, QERR |
| Блокировки | Кто ждёт, на каких областях, где в коде; TTIMEOUT с виновниками, TDEADLOCK с графом |
| Ошибки | Тренды QERR/EXCP, топ исключений, последние ошибки с контекстом |

## Структура репозитория

| Путь | Что |
|------|-----|
| `cpp_parse/` | **Ядро-нормализатор** (события ТЖ → NDJSON). Будет вынесено в `core/` как библиотека |
| `cpp/` | Старая версия — счётчик уникальных Context (донор quote-parity логики) |
| `agents/go/` | Go-участник bake-off (проходит golden-гейт) |
| `agents/{rust,cpp}/` | будущие участники |
| `deploy/` | docker-compose (ClickHouse + Grafana + Prometheus), схема БД, импортёр, дашборды |
| `bench/` | скрипты замеров (корпуса-junctions, прогрев, wall/CPU/RSS) |
| `tests/golden/` | byte-exact тесты нормализатора: 19 кейсов + раннер |
| `docs/` | документация (см. [docs/README.md](docs/README.md)) |
| `ROADMAP.md` | план по фазам и текущий статус |
