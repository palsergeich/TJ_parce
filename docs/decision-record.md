## Вердикт: агент-сборщик пишем на Go

**Победитель — Go (90.2), C++ core — 88.2, Rust — 86.3.**

Оговорка о прочтении весов: протокол называет в tail-критерии «корректность + p95» — таблица
градуирует p95 (все прошли гейт корректности). При прочтении «tail = только pass/fail» суммы
становятся C++ 91.5 vs Go 90.2 — фотофиниш в обе стороны. Решающие практические аргументы за Go:

1. **Три из четырёх «агентских» критериев** (parse, RSS, tail-латентность) — за Go; C++ выигрывает
   только вставку, и этот разрыв — уровень драйвера (clickhouse-go), а не архитектуры: лечится
   колоночными блоками/пулом соединений, тогда как отставание C++ по tail-латентности заложено
   в его однопоточный follow-цикл.
2. **Операционная простота**: один статический бинарь, `go build`, кросс-платформенность из
   коробки (C++ sink — WinHTTP, Windows-only; сборка требует VS-генератора из-за кириллицы пути).
3. **Лучшая память на проде** (52 МБ на стресс-файле 5.76 ГБ) — целевая машина это сервер 1С,
   которому нельзя мешать.

## Роли после решения

- **Go (`agents/go`)** — продуктовый агент фазы 3: служба, конфиг, /metrics, деплой.
- **C++ (`core/`)** — остаётся **эталонным нормализатором и библиотекой** (golden-истина форматной
  спеки, C ABI для будущих встраиваний, самый быстрый batch-путь для разовых массовых импортов).
- **Rust (`agents/rust`)** — консервируется как независимая реализация спецификации
  (ценность: дифф-фаззинг три-стороннего согласия, доказательство самодостаточности спеки).
