# Серия после фикса памяти Go/Rust — 2026-07-06

Повтор официальных серий после устранения цельночитаемых файлов у Go и Rust
(чанковый ридер + байтовый бюджет допуска + стриминг головного файла — по образцу core, коммит фикса см. в истории).
Условия: Docker Desktop и WSL полностью погашены, тёплый кэш для medium, ротация, `--threads 8`.
Отклонение то же: без прав администратора (Defender-исключений нет) — первый раунд medium снова забракован по выбросу первого прогона Go (23.8 с при 3.1% CPU — скан свежесобранного exe), зачтён повтор в чистых условиях.

## Серия 1: bench-medium 5.41 ГБ, тёплый кэш, медианы из 3 (ротация)

| Участник | Медиана, с | МБ/с | Разброс | Peak RSS, МБ |
|---|---:|---:|---:|---:|
| **Go** | **1.799** | **3077** | 1.4% | 905 |
| C++ core | 1.840 | 3009 | 3.3% | 538 |
| Rust | 2.418 | 2289 | 2.4% | 682 |

После фикса **Go ускорился** (2738 → 3077: меньше GC-давления с переиспользуемыми буферами) и обошёл C++ (разница 2.2% — на грани шума, но стабильна по прогонам). **Rust замедлился** (2614 → 2289): его новый чанковый сканер медленнее старого цельночитаемого пути — кандидат на оптимизацию (поиск \n SWAR vs memchr).

## Серия 2: стресс «гигантский файл» (5.76 + 1.86 ГБ, холодный кэш)

| Участник | Wall, с | Peak RSS, МБ | Было RSS (04.07) |
|---|---:|---:|---:|
| Go | 9.7 | **51.8** | 16 843 |
| Rust | 10.7 | **70.7** | 14 963 |
| C++ core | 12.7 | 2 368 | 2 350 |

Фикс подтверждён: **уменьшение памяти в ~300 раз**, без потери скорости (диск-баунд). Неожиданный поворот: байтовый бюджет агентов оказался **строже родного решения core** — у ядра окно допуска считает файлы (2×workers), и второй файл коллекции буферизуется целиком (2.4 ГБ). Задача: бэкпорт байтового бюджета в core.

## Серия 3: полный корпус 175 ГБ, холодный, по одному прогону

| Участник | Wall, с | ГБ/с | Peak RSS | События |
|---|---:|---:|---:|---:|
| Rust (бежал 1-м, самый холодный) | 210.2 | 0.77 | 1.0 ГБ | 121 485 348 |
| C++ core (2-м, частично прогрет) | 75.3 | 2.16 | 32 ГБ* | 121 485 348 |
| Go (3-м) | 157.0 | 1.04 | 1.9 ГБ | 121 485 348 |

**Кросс-сравнение скоростей в этой серии некорректно**: без сброса page cache порядок запуска даёт следующим участникам до ~18% корпуса из RAM. Цель серии — устойчивость и точность: все трое дошли без ошибок, счётчики совпали до единицы. RSS Go/Rust на полном корпусе теперь жёстко ограничен (1–1.9 ГБ против прежних 43–44).
\* WorkingSet C++ включает file-backed страницы mmap — это кэш ОС, не приватные аллокации.
