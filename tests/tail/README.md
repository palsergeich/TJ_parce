# tests/tail — харнесс tail-режима (bakeoff-protocol.md §4.5)

Проверяет `--follow` всех участников bake-off против живого ClickHouse
(контейнер `tj-clickhouse`, таблица `tj_bench.events`, HTTP `localhost:8123`).

## Запуск

```powershell
# зачётный прогон (все сценарии, ~5 минут)
.\run_tail.ps1 -Agent ..\..\agents\go\tj-agent-go.exe

# отладка: укороченные длительности / отдельные сценарии / не удалять данные
.\run_tail.ps1 -Agent <exe> -Quick -Only S2,S4 -KeepData
```

Прочее: `-Dsn` — значение `--sink` агента (по умолчанию `clickhouse`);
`-OutJson` — путь JSON-отчёта (по умолчанию `%TEMP%\last_run_<агент>.json`);
`-AgentThreads N` — добавить `--threads N`. Exit-код: 0 — всё PASS, 1 — есть FAIL,
2 — среда не готова. Каждый сценарий начинается с `TRUNCATE tj_bench.events` и
свежих каталогов логов/состояния в `%TEMP%\tj_tail_*` (при FAIL или `-KeepData`
каталоги сохраняются вместе с stdout/stderr агента).

## Контракт --follow (кратко)

```
<exe> --follow --input <dir> --sink clickhouse[:<dsn>] --state <dir> --stop-file <path>
      [--poll-ms 500] [--idle-close-ms 2000] [--threads N] [--stats-json <path>]
```

Первичный скан существующих файлов (с учётом чекпоинтов) → tail; подхват нового
файла < 2 с; файлы открываются с полным share (writer 1С держит их открытыми);
событие закрывается следующей строкой-маской `^\d{2}:\d{2}\.\d{6}-\d+,`, ЛИБО по
idle >= `idle-close-ms`, когда накопленный хвост оканчивается `\n`, ЛИБО дренажом
на stop; незавершённая строка (без `\n`) не эмитится никогда; чекпоинт на файл
(identity = volume serial + file index) двигается только после подтверждённой
вставки в CH (min-contiguous) и пишется атомарно в `--state`; усечение
(size < offset) или смена identity → чтение с 0; после crash допустим
at-least-once (ограниченные дубли), потери запрещены; по появлению `--stop-file` —
дренаж и exit 0; порог MIN_FILE_SIZE=100 перепроверяется по мере роста файла.

## Сценарии и их проверки

| # | Что | Assert |
|---|-----|--------|
| S1 | 30 с @ 10 000 соб/с, стоп генератора, 10 с дренаж, stop-file | потерь 0 (`uniqExact(props['Usr'])` == записано); `count()` == max(counter); дублей 0 (допуск ≤ 0.01% только при заявленном at-least-once); graceful exit 0 |
| S2 | хвост без `\n`, дозапись через 5 с | обрезок не эмитится (ждём > idle-close); после дописи ровно 2 новые записи; значение свойства склеено через границу дозаписи байт-в-байт |
| S3 | ротация каждые 10 с, 25 с записи | события всех файлов в CH; счётчики без потерь и непрерывны на границах файлов (подхват нового файла без пропуска) |
| S4 | truncate до 0 живьём + 500 новых с маркером `trunc_` | агент не упал; новые 500 дошли (рестарт с offset 0); старая волна не вставлена повторно |
| S5 | `Stop-Process -Force` на 10-й секунде, рестарт с тем же `--state` | потерь 0; дубли отчётно (at-least-once допустим, ждём «мало»); рестартованный агент выходит gracefully |
| S6 | 30 с @ 20 000 соб/с, опрос каждые 2 с | p95(now − max(GenMs)) < 5000 мс; контроль полноты после дренажа |
| S7 | sharing (неявно в S1: генератор держит файл открытым с FileShare.Read весь прогон) | S1 прошёл; stderr агента без ошибок |

## Генератор (`generator.ps1`)

```powershell
.\generator.ps1 -Dir <кат> -Rate 10000 -DurationSec 30 [-StartCounter 1] [-RotateAfterSec 10] [-MarkerPrefix tail_]
```

Пишет `<Dir>\rphost_1\<YYMMDDHH>.log` (BOM в начале файла, CRLF), держит файл
открытым .NET StreamWriter'ом с `FileShare.Read` весь прогон, пачки по тику
100 мс + Flush. Событие: `MM:SS.ssssss-<dur>,CALL,1,Usr=<префикс><counter>,GenMs=<unix_ms>,Memory=<n>`;
каждое 500-е несёт многострочный `Context='...'` на 3 строки. Повторный запуск в
тот же каталог/час пересоздаёт файл (FileMode.Create → усечение, используется S4);
`-RotateAfterSec N` каждые N секунд закрывает файл и открывает файл следующего
часа. Сайдкар `<Dir>\timeline.csv` — `counter,unix_ms` каждого 1000-го события.
stdout — одна строка JSON: `{"total":..,"last":..,"actual_rate":..,"files":[..]}`.
Горячий цикл — встроенный C# (Add-Type): устойчиво держит 100k+ соб/с;
чисто-PowerShell цикл 20k соб/с не тянет.

Все `.ps1` сохранены в UTF-8 **с BOM** (правило репозитория для PS 5.1).
